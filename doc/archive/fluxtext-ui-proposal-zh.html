FluxText — UI 设计提案
基于源码一致的现状 V2 版本进行的彻底重设计。 新方向：深色框架 + 白色 Monaco 画布。保留所有框架和插件界面。 没有凭空捏造的功能，没有营销文案，没有过度简化的结构。
提案 / 2026-06-10
1 · 现状问题诊断
层级坍塌
顶部栏 (26px)、窗格标签 (24px)、窗格状态 (25px) 和状态栏 (24px) 视觉比重完全相同。缺少清晰的框架与内容区分。
侧边栏辨识度
字母缩写 (E, P, S) 只有背景填充的激活状态。没有左侧高亮条。固定操作槽不可预测地打破了导航/设置的垂直节奏。
窗格辨识度
窗格标签仅显示数字和关闭按钮。活动窗格无法一眼分辨。窗格类型（纯文本、渲染器等）被隐藏在状态栏中。
面板项比重
框架命令和插件命令使用完全相同的图标处理。除非仔细阅读副标题，否则无法看出命令是来自插件的。
插件卡片扁平化
无论状态如何（激活、错误、禁用），所有插件卡片的视觉比重都一样。错误卡片只能通过边框颜色变化来区分；状态信号很弱。
运行器头部拥挤
Auto 徽章、运行、控件、重置和取消固定都在一行中具有相同的视觉比重。主要操作（运行）在视觉上不占主导地位。
设置双列未对齐
卡片高度根据包含的行数而变化，产生参差不齐的垂直节奏。每个卡片标题的突出首字母感觉像装饰，没有增加导航价值。
亮色模式视觉疲劳
全白 + 极浅灰色的配色在框架和编辑器画布之间没有提供深度提示。在没有区分的白色界面上长时间工作容易疲劳。

插件 UI 影响映射 插件触及哪些界面 · 仅限当前
命令面板
所有命令都是插件入口。行暴露了固定、参数、输入回退和最近使用情况。插件命令获得带有强调色色调的图标（与框架命令在视觉上区分开来）。
即时建议
计算器提供带有复制操作的实时结果行（例如 1+2*3 = 7）。显示在常规命令上方，带有强调色的左侧栏。
工作区渲染器
Text Diff 拥有并排的 Monaco 渲染器和 JSON 语义工具栏。框架承载界面；插件提供内容和工具栏行。
面板 / 工具栏
插件提供右上角工具栏按钮和 V2 面板。正则测试器和 JS 过滤器显示为插件命令界面，而不是默认的框架界面。
固定的运行器
Pinnable commands become sidebar pinned-action slots and open the Input/Output runner. Controls bar uses plugin-contributed widgets when no custom panel is registered.
包管理器
插件生命周期卡片显示内置 · 已安装 · 开发状态栏，能力，源码查看器操作，切换，重新加载和删除。可见的格式错误包错误以及错误栏。

5 · 插件边界说明
框架拥有 — 不要提取到插件中
  • 插件注册表、命令注册表、渲染器注册表、面板注册表
  • 渲染器生命周期（挂载 / 卸载 / 焦点移交）
  • 工作区状态：窗格布局、活动窗格、拆分模式
  • 窗格文本读/写 API、选择、Monaco 实例生命周期、装饰桥接
  • 编辑器原语和侧边槽渲染
  • 设置 store and context provider
  • 副作用运行器，表面占用跟踪器
  • 应用外壳、侧边栏、顶部栏、状态栏、窗格网格、窗格标签 — 所有框架组件
  • 命令面板 host, 全局启动器 host
  • 固定的运行器 host (framework provides the Input/Output shell; plugin provides the command)
插件拥有 — 绝不能移入框架
  • 差异 API 和比较 API — 完全由插件拥有
  • DiffSurface、CompareRenderer、TextDiffRenderer — 插件组件
  • 并排差异布局、Monaco 差异实例 — 插件布局关注点
  • 语义差异切换、JSON 语义状态、数组比较控件
  • Esc / 退出差异对比 handling — plugin keyboard contract
  • 计算器即时建议逻辑和结果行内容
  • 正则测试器面板和工具栏贡献
  • JS 过滤器表达式栏
  • JSON ↔ YAML 转换逻辑
  • 插件特定的回退文本、显示文案、功能元数据
  • 在右上角工具栏贡献槽中呈现的插件工具栏按钮
差异 Monaco 窗格必须保持可编辑。 Text Diff 插件通过框架的宿主窗格更新 API 读取和写入窗格内容。不要将 Monaco 差异窗格转换为静态视觉预览。在将窗格状态传递给差异渲染器时，必须保留左右窗格上的 editable: true 标志。
